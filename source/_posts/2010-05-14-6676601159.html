--- layout: post title: Tomcat Character Encoding Configuration for UTF-8 ---

Tomcat is still a very popular servlet container, but the default character encoding for Tomcat is ISO-8859-1. This causes problem if you want your website to be international because you will need to UTF-8 encoding. There is an instruction on Tomcat wiki talking about how to set character encoding. I just want to add some extra points.

  • If you are using Spring framework, you can use the character filter provided by Spring.

 <!--
    - This is useful because current browsers typically do not
    - set a character encoding even if specified in the HTML page or form
 -->
 <filter>
    <filter-name>encodingFilter</filter-name>
    <filter-class>org.springframework.web.filter.CharacterEncodingFilter</filter-class>
    <init-param>
        <param-name>encoding</param-name>
        <param-value>UTF-8</param-value>
    </init-param>
    <init-param>
        <param-name>forceEncoding</param-name>
        <param-value>true</param-value>
    </init-param>
 </filter>

 <filter-mapping>
    <filter-name>encodingFilter</filter-name>
    <url-pattern>/*</url-pattern>
 </filter-mapping>

    • Make sure that the character filter is the FIRST in your filter chain. This is actually very important since filter chain does have order.
